PROMPT REEDISENO MOBILE 2026 — CLIENTE + DELIVERY (REPARTIDOR)

CONTEXTO:
LOGIFAST necesita un reediseño mobile-first para los roles de CLIENTE y REPARTIDOR (delivery). No quiero algo basico ni generico. Quiero interfaces que se sientan como las apps mas avanzadas de 2026: micro-interacciones fluidas, layouts no convencionales, profundidad visual con glassmorphism y layers, animaciones choreografiadas, y una experiencia que haga que el usuario diga "wow, esto se siente caro".

Referencias de nivel: Uber Eats 2026, Glovo Pro, Rappi Prime, Grab Superapp, DoorDash DashPass, Bolt Food, Getir, o cualquier app de delivery que este innovando en UX mobile.

Este prompt es PURAMENTE VISUAL Y DE INTERACCION. Define como se VEA y SE SIENTA cada pantalla, cada transicion, cada micro-interaccion. La logica ya esta construida en prompts anteriores.

Mismas tokens, paleta, tipografia. Nuevas reglas de diseno mobile explicitas.

DEPENDENCIAS ADICIONALES NECESARIAS:
- framer-motion (ya instalado) — para animaciones complejas
- react-leaflet (ya instalado) — para mapas
- haptic feedback: navigator.vibrate() nativo del browser
- CSS: scroll-snap, backdrop-filter, CSS containment, view transitions API

=====================================================
PRINCIPIOS DE DISENO MOBILE 2026
=====================================================

1. DEPTH LAYERS: la app tiene 3 capas de profundidad
   - Capa 0: mapa/fondo (siempre detras)
   - Capa 1: contenido principal (scrollable)
   - Capa 2: elementos flotantes (FAB, bottom nav, bottom sheet)
   Cada capa tiene su propio nivel de elevacion (sombra + blur)

2. GLASSMORPHISM INTELIGENTE: no es poner blur en todo. Es usar blur SOLO en elementos que flotan sobre contenido dinamico (mapa, imagenes, video). El contenido estatico NO necesita blur.

3. SNAP INTERACTIONS: todo lo que sube desde abajo (bottom sheets, keyboards, panels) tiene snap points. El usuario puede arrastrar a 3 posiciones: minimizado, medio, completo.

4. GESTURE-FIRST: la interfaz responde a gestos, no solo a taps. Swipe, pull, drag, long-press. Cada gesto tiene feedback visual Y haptic.

5. RHYTHM: las animaciones tienen ritmo. No todo se mueve al mismo tiempo. Hay un patron temporal: primero el header, luego el contenido, luego los detalles. Stagger inteligente.

6. SKEUOMORPHIC TOUCH: elementos tactiles se sienten fisicos. Botones que se "hunden" al tocar, cards que se "elevan" al arrastrar, switches que "saltan" al cambiar.
