      "Desactivar tienda" / "Activar tienda" (toggle)
      "Eliminar tienda" (modal destructivo)

VISTA GRID (alternativa):
- Tarjetas de tienda 280px ancho:
  Portada: area 120px con color de la tienda
  Logo: 48x48 circulo superpuesto
  Nombre + categoria + calificacion
  Stats: pedidos, ingresos
  Badge de estado
  Acciones rapidas: editar / productos / desactivar

MODAL CREAR/EDITAR TIENDA:
- Max-width 600px, scroll interno si es necesario
- Secciones:

  PASO 1 — Informacion basica:
    Nombre de la tienda: input
    Categoria: select con las 8 categorias
    Descripcion: textarea, max 300 chars
    Telefono: input
    Email: input
    Direccion: input con autocompletado de Managua (GET /api/direcciones)
    Mapa: al seleccionar direccion, marcador aparece en Leaflet, se puede ajustar arrastrando
    Lat/Lng: se llenan automaticamente desde la direccion (o manual)

  PASO 2 — Identidad visual:
    Color de portada: selector de color con presets (8 colores de la paleta + custom)
    Color del logo: selector de color
    Iniciales del logo: input 2 chars, preview en vivo del circulo con esas iniciales
    Preview del card de tienda: como se vera en la app del cliente

  PASO 3 — Operacion:
    Horario por dia: grid con Lun-Dom, selector de hora apertura y cierre por dia
    Toggle "Abierto ahora" por dia
    "Copiar horario a todos los dias" boton
    Zonas de cobertura: checkboxes de las zonas de Managua (Centro, Villa Fontana, Los Robles, Bello Horizonte, Monseñor Lezcano, Reparto Schick, Masaya)

  PASO 4 — Configuracion:
    Estado: "Activa" / "Inactiva" / "Cerrada"
    "Destacada" toggle: si esta activa, aparece en "Populares cerca de ti" en el feed del cliente
    "Verificada" toggle: muestra badge azul de verificada
    "Popular" toggle: muestra badge naranja
    Propietario (opcional): select de usuarios si se quiere asignar un dueño de tienda

11B. EDITOR DE FEED (ESTILO PEDIDOYA)
=====================================================

ESTE ES EL MODULO MAS IMPORTANTE DE ESTE PROMPT. El admin controla COMPLETAMENTE lo que el cliente ve en su pantalla de Inicio y Explorar, como un editor de contenido de PedidosYa.

SUB-SECCIONES DEL EDITOR:

11B.1 — SECCIONES DEL FEED:

El feed del cliente se compone de SECCIONES que el admin puede crear, ordenar, activar/desactivar. Cada seccion es un bloque vertical en la pantalla del cliente.

LISTA DE SECCIONES DEL FEED:
- Cada seccion como item draggable (drag & drop para reordenar):
  - Handle de arrastre (6 puntos verticales)
  - Nombre de la seccion 15px bold
  - Tipo de seccion: badge
  - Cantidad de items dentro
  - Estado: toggle activar/desactivar
  - Vista previa: mini thumbnail de como se ve
  - Acciones: editar, duplicar, eliminar

SECCIONES DISPONIBLES (el admin puede crear multiples instancias de cada tipo):

TIPO 1 — "Carrusel de Banners":
- Nombre editable: "Promos del dia" / "Ofertas destacadas" / etc.
- Layout en el cliente: scroll horizontal de banners full-width
- Contenido: los banners que el admin ya creo en Marketing v3
- Configuracion: max items visibles, auto-slide on/off, intervalo de slide

TIPO 2 — "Categorias de Tiendas":
- Nombre editable: "Explora por categoria"
- Layout en el cliente: scroll horizontal de pills/iconos de categoria
- Contenido: categorias de tiendas con iconos
- Configuracion: que categorias mostrar, orden

TIPO 3 — "Tiendas Destacadas":
- Nombre editable: "Populares cerca de ti" / "Nuevas tiendas" / "Tus favoritas"
- Layout en el cliente: scroll horizontal de tarjetas de tienda
- Contenido: seleccion de tiendas (manual o automatica)
- Configuracion:
  Fuente: "Manual" (admin selecciona tiendas) / "Automatica" (por calificacion, pedidos, recientes)
  Max items: 5-20
  Mostrar: calificacion, tiempo, envio, badges

TIPO 4 — "Productos Destacados":
- Nombre editable: "Lo mas pedido" / "Novedades" / "Ofertas especiales"
- Layout en el cliente: scroll horizontal de tarjetas de producto
- Contenido: seleccion de productos de cualquier tienda
- Configuracion:
  Fuente: "Manual" / "Mas vendidos" / "Mejor calificados" / "Con descuento"
  Max items
  Mostrar: precio, tienda, descuento

TIPO 5 — "Ofertas Relampago":
- Nombre editable: "Ofertas de hoy" / "Descuentos exclusivos"
- Layout en el cliente: tarjeta grande con timer de cuenta regresiva
- Contenido: producto o tienda con oferta limitada + porcentaje de descuento + tiempo restante
- Configuracion: producto/tienda, descuento %, duracion de la oferta, codigo asociado

TIPO 6 — "Feed de Contenido":
- Nombre editable: "Novedades" / "Avisos importantes"
- Layout en el cliente: lista vertical de items (anuncios, promociones, novedades)
- Contenido: los items de feed que el admin creo en Marketing v3
- Configuracion: max items visibles, orden (cronologico o manual)

TIPO 7 — "Recomendaciones Personalizadas":
- Nombre editable: "Para ti" / "Basado en tus pedidos"
- Layout en el cliente: mixto (tiendas + productos basados en historial)
- Contenido: automatico, algoritmo simple basado en historial del cliente
- Configuracion: mostrar tiendas, productos, o ambos

TIPO 8 — "Separador Visual":
- Nombre editable: cualquier texto
- Layout en el cliente: linea horizontal con texto centrado (ej: "—— Populares ——")
- Configuracion: texto, estilo (linea, puntos, gradiente)

TIPO 9 — "Acceso Rapido Dual":
- Nombre editable: "Enviar o Comprar"
- Layout en el cliente: las 2 tarjetas lado a lado (Enviar paquete / Comprar y recibir)
- Configuracion: textos, colores, iconos de cada tarjeta

TIPO 10 — "Buscador":
- Nombre editable: "Buscar"
- Layout en el cliente: barra de busqueda prominente
- Configuracion: placeholder del input

EDITOR DE SECCION (al crear/editar):
- Panel lateral (desktop) o pantalla (mobile)
- Campos:
  Nombre de la seccion: input
  Tipo de seccion: selector visual con thumbnails de cada tipo
  Configuracion especifica segun tipo (campos dinamicos)
  Visibilidad: "Mostrar a todos" / "Solo clientes nuevos" / "Solo clientes frecuentes" / "Solo clientes VIP"
  Programacion: "Visible ahora" / "Desde fecha" / "Hasta fecha" / "Permanente"
- Preview en tiempo real: a la derecha del editor, un mockup de telefono mostrando como se vera esa seccion

VISTA PREVIA DEL FEED COMPLETO:
- Panel o modal que muestra un mockup de telefono (320px ancho) con TODO el feed del cliente renderizado
- Las secciones aparecen en el orden configurado
- Se puede scroll vertical dentro del mockup
- Actualizado en tiempo real mientras el admin edita
- Boton "Vista previa" siempre visible en la esquina

   - Linea + puntos

2. Pedidos por tienda (BarChart horizontal):
   - Top 10 tiendas por cantidad de pedidos
   - Barras naranjas con nombre de tienda a la izquierda
   - Valor a la derecha

3. Categorias mas vendidas (DonutChart):
   - Porcentaje de pedidos por categoria
   - Centro: total de pedidos
   - Leyenda con colores

4. Tiempo promedio de entrega por tienda (BarChart):
   - 10 tiendas
   - Barras con gradiente de color: verde (<20min), amarillo (20-35min), rojo (>35min)
   - Linea de referencia: "Objetivo: 30 min"

5. Productos mas vendidos (BarChart horizontal):
   - Top 10 productos por cantidad vendida
   - Con nombre de tienda al lado

6. Mapa de calor de pedidos:
   - Leaflet con circulos proporcionales a la cantidad de pedidos por zona
   - Zonas mas activas: circulos mas grandes y mas oscuros

   - 24 barras (una por hora del dia)
   - Muestra a que hora hay mas pedidos
   - Barras naranjas, las horas pico mas oscuras

8. Conversion del feed:
   - "Impresiones de banner" → "Clicks" → "Pedidos"

TABLA DE RANKING:
- Top tiendas: nombre, pedidos, ventas, calificacion, tiempo promedio

model FeedSeccion {

=====================================================
IMPORTANTE
=====================================================

1. El Editor de Feed es la herramienta MAS poderosa del admin para el marketplace. Controla cada pixel que el cliente ve. Sin tocar codigo.
2. Las secciones del feed son modulares — el admin puede crear cualquier combinacion de bloques.
3. Las ofertas relampago crean urgencia — timer visible, descuento grande, tiempo limitado.
4. Los banners ahora pueden enlazar a tiendas y productos, no solo a envios.
5. La analitica del marketplace separa metricas de envios vs compras — el admin ve ambos mundos.
6. Las comisiones por tienda son configurables — el admin controla la monetizacion.
7. Los pedidos de compra tienen un flujo de estados diferente a los envios (Recibido → Preparando → Listo → En camino → Entregado).
8. El feed se adapta por segmento: un cliente nuevo ve un feed, un cliente frecuente ve otro.
9. Las plantillas de feed permiten cambiar toda la experiencia del cliente en 1 click (ej: feed navidad).
10. Todo se conecta: Marketing crea banners/codigos → Feed los posiciona → Cliente los ve → Compra → Admin monitorea → Analitica mide.
11. Las tiendas con baja calificacion o alto tiempo de entrega aparecen en "Peores performers" — el admin puede intervenir.
12. El marketplace funciona como PedidosYa: el cliente descubre, compra, rastrea, califica. El admin controla toda la experiencia.
